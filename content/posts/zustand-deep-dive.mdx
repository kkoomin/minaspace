기본적으로 Zustand는 셀렉터 구조로 state를 가져와 사용한다. 하지만 스토어가 많이 쪼개져 있는 실무 프로젝트 구조에서는, 스토어에서 state와 action을 가져오는 코드가 너무 길어진다. 이를 개선할 수 있는 방법을 정리했다.

## 1. Auto Generating Selectors

Zustand 공식 문서에서 제안하는 구조로, store에서 import해야 하는 것이 많아질 경우 selector 구조를 반복적으로 작성해야 하는 문제를 개선한다.

```typescript
import { StoreApi, UseBoundStore } from "zustand";

type WithSelectors<S> = S extends { getState: () => infer T }
  ? S & { use: { [K in keyof T]: () => T[K] } }
  : never;

const createSelectors = <S extends UseBoundStore<StoreApi<object>>>(
  _store: S,
) => {
  let store = _store as WithSelectors<typeof _store>;
  store.use = {};
  for (let k of Object.keys(store.getState())) {
    (store.use as any)[k] = () => store((s) => s[k as keyof typeof s]);
  }

  return store;
};
```

기존 Zustand에서 사용하는 방식대로 스토어를 `create()`해서 구성하고, 해당 스토어를 `createSelectors`로 묶어서 export한다.

```typescript
export const useOrderStore = createSelectors(
  create<IOrderState>()(
    devtools(
      (set, get) => ({
        orderType: null,
        isBid: false,
        completed: false,

        setOrderType: (orderType: OrderType) => set({ orderType }),
        setIsBid: (isBid: boolean) => set({ isBid }),
        setCompleted: (completed: boolean) => set({ completed }),
      }),
      { name: "useOrderStore" },
    ),
  ),
);
```

이렇게 하면 컴포넌트에서 아래처럼 간단하게 state나 action을 가져올 수 있다.

```typescript
const isBid = useOrderStore.use.isBid();
const setIsBid = useOrderStore.use.setIsBid();
```

## 2. Actions를 한 번에 묶어서 export

Actions는 스토어의 값을 업데이트하는 함수다. 기존 방식에서는 여러 action을 꺼낼 때 아래처럼 `useShallow`를 써야 했다.

```typescript
// ❌ 모든 state가 변경될 때마다 컴포넌트를 업데이트함
const store = useOrderProductStore();

// ✅ useShallow로 여러 개의 action 가져오기 (객체 형태)
const { setName, setThumb, setBrandBgColor } = useOrderProductStore(
  useShallow((state) => ({
    setName: state.setName,
    setThumb: state.setThumb,
    setBrandBgColor: state.setBrandBgColor,
  })),
);
```

Actions는 정적이고 변경되지 않기 때문에, 별도의 `actions` 객체로 묶어서 하나의 훅으로 export하는 방법을 쓸 수 있다.

```typescript
const useOrderProductStore = createSelectors(
  create<OrderProductState>()(
    devtools(
      (set) => ({
        name: "",
        thumb: "",
        brandBgColor: "",
        size: "",
        inventoryId: 0,
        productGrade: PRODUCT_GRADE_TRADE_CODE,

        actions: {
          setName: (name: string) => set({ name }),
          setThumb: (thumb: string) => set({ thumb }),
          setBrandBgColor: (brandBgColor: string) => set({ brandBgColor }),
          setSize: (size: string) => set({ size }),
          setInventoryId: (inventoryId: number) => set({ inventoryId }),
          setProductGrade: (productGrade) => set({ productGrade }),
        },
      }),
      { name: "useOrderProductStore" },
    ),
  ),
);

// ✅ actions selector를 하나의 훅으로 export
export const useOrderProductStoreActions = () =>
  useOrderProductStore((state) => state.actions);
```

컴포넌트에서는 구조분해 할당으로 깔끔하게 사용할 수 있다.

```typescript
const { setName, setThumb, setBrandBgColor, setSize, setProductGrade } =
  useOrderProductStoreActions();
```

## 3. Flux 패턴 (Redux-like)

Redux와 유사하게 reducer 패턴을 사용하고 싶은 경우, store의 최상단에 `dispatch` 함수를 정의할 수 있다.

```typescript
const types = { increase: "INCREASE", decrease: "DECREASE" };

const reducer = (state, { type, by = 1 }) => {
  switch (type) {
    case types.increase:
      return { grumpiness: state.grumpiness + by };
    case types.decrease:
      return { grumpiness: state.grumpiness - by };
  }
};

const useGrumpyStore = create((set) => ({
  grumpiness: 0,
  dispatch: (args) => set((state) => reducer(state, args)),
}));

const dispatch = useGrumpyStore((state) => state.dispatch);
dispatch({ type: types.increase, by: 2 });
```

Redux를 쓰던 팀이 Zustand로 마이그레이션할 때 유용한 패턴이다. 다만 Zustand의 단순함을 살리려면 굳이 flux 패턴을 도입하지 않는 쪽이 낫다고 생각한다.
