중고상품 거래 서비스를 개발하면서 Zustand 스토어가 점점 비대해졌습니다. 하나의 파일에 상품, 유저, 필터 상태가 섞이자 유지보수가 어려워졌고, 결국 도메인 단위 분리를 결정했습니다.

## 문제 정의

초기에는 `useAppStore` 하나로 모든 상태를 관리했습니다.

```ts
const useAppStore = create<AppState>((set) => ({
  products: [],
  user: null,
  filters: { category: "all", priceRange: [0, 100000] },
  setProducts: (products) => set({ products }),
  setUser: (user) => set({ user }),
  setFilters: (filters) => set({ filters }),
}));
```

문제는 `user`만 필요한 컴포넌트도 `products`가 바뀔 때마다 리렌더링됐다는 점입니다.

## 기술 구현

세 개의 독립적인 스토어로 분리했습니다.

```ts
// store/useProductStore.ts
const useProductStore = create<ProductState>((set) => ({
  products: [],
  setProducts: (products) => set({ products }),
}));

// store/useUserStore.ts
const useUserStore = create<UserState>((set) => ({
  user: null,
  setUser: (user) => set({ user }),
}));

// store/useFilterStore.ts
const useFilterStore = create<FilterState>((set) => ({
  filters: { category: "all", priceRange: [0, 100000] },
  setFilters: (filters) => set({ filters }),
}));
```

### useShallow 적용

같은 스토어에서 여러 값을 가져올 때는 `useShallow`로 불필요한 리렌더를 방지했습니다.

```ts
import { useShallow } from "zustand/react/shallow";

const { products, setProducts } = useProductStore(
  useShallow((s) => ({ products: s.products, setProducts: s.setProducts })),
);
```

## Auto Selectors 패턴

반복되는 셀렉터 작성을 줄이기 위해 `createSelectors` 헬퍼를 만들었습니다.

```ts
function createSelectors<S extends UseBoundStore<StoreApi<object>>>(store: S) {
  const selectors = store as WithSelectors<typeof store>
  for (const key of Object.keys(store.getState())) {
    selectors.use = selectors.use || {}
    ;(selectors.use as Record<string, unknown>)[key] = () =>
      store((s) => s[key as keyof typeof s])
  }
  return selectors
}

const useProductStore = createSelectors(create<ProductState>()(...))

// 사용
const products = useProductStore.use.products()
```

## 결과

- 불필요한 리렌더 제거로 상품 목록 페이지 렌더링 횟수 약 40% 감소
- 각 스토어가 단일 책임을 가져 테스트 작성이 쉬워짐
- 새 도메인 추가 시 기존 스토어 영향 없이 독립적으로 개발 가능
